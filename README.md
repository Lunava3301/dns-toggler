# DNS Toggler 🚀

Удобный интерактивный кроссплатформенный консольный инструмент для быстрого переключения DNS-серверов в один клик. Работает на macOS и Windows из единого файла.

## ✨ Особенности

*   🎨 **Красивый и современный интерфейс**: Мягкая 256-цветная палитра ANSI (пастельные тона) и быстрые анимации загрузки (спиннер оптимизирован до 1 секунды).
*   📦 **Единый код (`dns_toggler.py`)**: Вся логика для macOS и Windows объединена в один кроссплатформенный файл.
*   🔒 **Автоматическое повышение прав**:
    *   **macOS**: При запуске автоматически проверяет права доступа и запрашивает права суперпользователя (`sudo`) прямо в окне терминала, если это необходимо.
    *   **Windows**: Автоматически запрашивает права администратора через окно контроля учетных записей (UAC).
*   🌐 **Динамическое определение активного интерфейса**:
    *   Скрипт автоматически определяет текущий активный сетевой адаптер (будь то Wi-Fi, Ethernet или проводное соединение через USB-хаб) как на Windows, так и на macOS.
*   ⚡ **Предустановленные DNS-шаблоны**:
    *   Google DNS (`8.8.8.8`, `8.8.4.4`)
    *   Cloudflare DNS (`1.1.1.1`, `1.0.0.1`)
    *   AdGuard DNS (блокировка рекламы: `94.140.14.14`, `94.140.15.15`)
    *   Yandex DNS (безопасный/фильтрация: `77.88.8.8`, `77.88.8.1`)
*   🔧 **Валидация IP-адресов**: При вводе кастомных DNS-серверов выполняется проверка на корректность формата IP-адреса, предотвращая случайную поломку интернет-соединения из-за опечаток.

## 🛠️ Установка и использование

### macOS

> [!WARNING]
> При запуске скрипт автоматически запросит ваш системный пароль администратора в терминале через `sudo` для внесения изменений в настройки сети.

#### Способ 1: Запуск в один клик с рабочего стола (Toggle_DNS.command)
В репозитории находится вспомогательный файл `Toggle_DNS.command`. Вы можете скопировать его в любую папку или на Рабочий стол. При его запуске автоматически откроется Терминал и запустится основной скрипт `dns_toggler.py`.

Чтобы система разрешила запуск файла `.command`, выполните один раз в терминале:
```bash
chmod +x Toggle_DNS.command
```

#### Способ 2: Запуск вручную через Терминал
1. Перейдите в директорию проекта:
   ```bash
   cd ~/dns-toggler
   ```
2. Убедитесь, что скрипт имеет права на запуск:
   ```bash
   chmod +x dns_toggler.py
   ```
3. Запустите его:
   ```bash
   ./dns_toggler.py
   ```

---

### Windows

Скрипт автоматически запрашивает UAC-диалог для предоставления прав администратора.

#### Способ 1: Запуск в один клик (Toggle_DNS.bat)
В репозитории есть файл `Toggle_DNS.bat`. Вы можете скопировать его в любую папку или на Рабочий стол. При его запуске автоматически запустится `dns_toggler.py` через установленный в системе Python.

#### Способ 2: Запуск вручную
1. Запустите скрипт через интерпретатор Python:
   ```cmd
   python dns_toggler.py
   ```
2. Подтвердите запрос прав администратора в появившемся окне UAC.

## 📂 Структура проекта

*   `dns_toggler.py` — единый кроссплатформенный скрипт (macOS и Windows).
*   `Toggle_DNS.command` — переносимый ярлык быстрого запуска для macOS.
*   `Toggle_DNS.bat` — переносимый ярлык быстрого запуска для Windows.
*   `.gitignore` — исключение временных файлов Python и локального конфигурационного файла `.dns_config`.
